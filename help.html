Getting Started

Dash Humanizer automatically transforms em-dashes (‚Äî) and optionally en-dashes (‚Äì) into ellipsis (‚Ä¶) with proper spacing, making your writing feel more natural and conversational.

Quick Start

  1. Install the extension from the Chrome Web Store or load it unpacked
  2. Click the extension icon in your toolbar to verify it's enabled (should show "ON")
  3. Select any text containing dashes on a webpage
  4. A bubble will appear showing the humanized version
  5. Click "Copy" or "Replace Selection" to use the transformed text

Frequently Asked Questions

Why use ellipsis instead of em-dashes?

Ellipsis (‚Ä¶) creates a more conversational, natural tone in writing. Em-dashes can feel formal or abrupt, while ellipsis suggests a pause or trailing thought that feels more human and relatable.

Will this break my numeric ranges like "2019‚Äì2023"?

No! The extension intelligently detects numeric ranges and preserves them. It only transforms dashes used as clause breaks in sentences.

Does this work in Google Docs?

Yes! The extension fully supports Google Docs, including live typing transformation and proper undo/redo functionality.

Is my data being sent anywhere?

Absolutely not. All text processing happens locally in your browser. The extension makes zero network requests and collects no data. Your privacy is guaranteed.

Can I disable the automatic bubble?

Yes! Go to Settings and uncheck "Auto-humanize on selection". You can still use the context menu (right-click) to transform text manually.

Why isn't live typing working?

Make sure "Auto-humanize as I type/paste" is enabled in Settings. Also verify the extension is enabled (click the icon to check). If it still doesn't work, try refreshing the page.

What's the difference between em-dash and en-dash?

Em-dash (‚Äî) is longer and typically used for clause breaks. En-dash (‚Äì) is shorter and typically used for ranges. By default, the extension only transforms em-dashes, but you can enable en-dash transformation in Settings.

Can I use this on mobile?

Currently, this is a Chrome desktop extension. Mobile browser extensions have limited support, but we may explore mobile options in the future.

Will this work in code editors or terminals?

The extension automatically skips transformation in code blocks (<pre>, <code> tags) to preserve your code. However, it may not detect all specialized code editors. You can disable the extension temporarily if needed.

How do I type an actual em-dash if I want one?

If you have live typing enabled and want to keep an em-dash, you can:

  • Disable "Auto-humanize as I type/paste" in Settings
  • Use the extension's toggle to temporarily disable it
  • Type the dash and immediately undo (Ctrl+Z / Cmd+Z)

Does this sync across my devices?

Yes! Your settings are synced across all Chrome browsers where you're signed in, thanks to Chrome's sync storage.

What permissions does this extension need?

The extension requires:

  • contextMenus: For the right-click "Humanize dashes" option
  • storage: To save your settings
  • activeTab: To access and modify text on the current page
  • <all_urls>: To work on any website you visit
All permissions are necessary for core functionality. See the Privacy & Security section for details.

Troubleshooting

Selection bubble doesn't appear

Possible causes:

  • Extension is disabled - Click the icon and check if it shows "OFF"
  • "Auto-humanize on selection" is disabled in Settings
  • Selected text doesn't contain any dashes
  • Page hasn't fully loaded - Wait a moment and try again

Solution: Enable the extension and the setting, then try selecting text with dashes again.

Live typing isn't working

Possible causes:

  • "Auto-humanize as I type/paste" is disabled in Settings
  • Extension was installed after the page loaded
  • The field is in an iframe with cross-origin restrictions

Solution: Enable the setting and refresh the page. If still not working, try disabling and re-enabling the extension.

"Replace Selection" button doesn't work

Possible causes:

  • Text is in a read-only field (use "Copy" instead)
  • Complex rich text editor that the extension can't modify
  • Cross-origin iframe restrictions

Solution: Use the "Copy" button and paste manually, or use the context menu instead.

Clipboard copy fails

Possible causes:

  • Browser denied clipboard permission
  • Page security policy blocks clipboard access

Solution: Grant clipboard permission when prompted. If that doesn't work, the extension will show a modal where you can manually copy the text.

Extension slows down typing

Possible causes:

  • Very large text field (50,000+ characters)
  • Other extensions conflicting
  • Browser performance issues

Solution: Disable "Auto-humanize as I type/paste" for large documents. Use the selection bubble or context menu instead.

Settings don't save

Possible causes:

  • Chrome sync is disabled
  • Storage quota exceeded (very rare)

Solution: Check Chrome sync settings. Try disabling and re-enabling the extension.

Extension doesn't work on a specific site

Possible causes:

  • Site uses a complex custom editor
  • Site has strict Content Security Policy
  • Site loads content dynamically after page load

Solution: Try refreshing the page. If it still doesn't work, use the copy/paste workflow instead of live typing.
